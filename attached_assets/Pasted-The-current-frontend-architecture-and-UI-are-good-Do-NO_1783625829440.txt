The current frontend architecture and UI are good.

Do NOT redesign or modify the existing Owner, Receptionist, or Super Admin dashboards unless absolutely necessary.

Instead, extend the current application by adding a complete Student Portal while maintaining the same design language, component library, spacing, typography, colors, and overall UI consistency.

The Student Portal should feel like it belongs to the same application.

=========================================================
STUDENT PORTAL (NEW ROLE)
=========================================================

Add a new role called "Student".

Students should have their own authentication flow and dashboard.

The student portal should be responsive and work well on both desktop and mobile browsers.

The design philosophy should remain:

• Clean
• Minimal
• Modern SaaS
• Fast
• Premium
• No visual clutter
